# <h1 id="stack-web">Les différentes stacks Web (MERN, MEAN, LAMP...)</h1>

* **Base de données** : MySQL, MariaDB, PostgreSQL, MongoDB...
* **Serveur web** : Nginx, Apache, Tomcat, Express, IIS...
* **Langage côté client** : React (ou Next.js), Angular, Vue.js...
* **Langage côté serveur** : PHP, Node.js, ASP.NET, Python (Flask/Django)...

**Exemples de stacks populaires :**

| Stack | Description                         |
| ----- | ----------------------------------- |
| MERN  | MongoDB, Express, React, Node.js    |
| MEAN  | MongoDB, Express, Angular, Node.js  |
| LAMP  | Linux, Apache, MySQL, PHP           |
| MAMP  | macOS, Apache, MySQL, PHP           |
| WAMP  | Windows, Apache, MySQL, PHP         |
| WEMP  | Windows, Nginx (Engine), MySQL, PHP |

# <h1 id="etape7-fonction-classe">Étape 7 – Transformer un composant fonction en classe</h1>

### Version fonction :

```jsx
const App = () => {
  return <h1>Bonjour</h1>;
}
```

### Version classe :

```jsx
class App extends Component {
  render() {
    return <h1>Bonjour</h1>;
  }
}
```

# <h1 id="etape10-props-children">Étape 10 – Travailler avec `props.children`</h1>

### Appel :

```jsx
<College categorie="public" endroit="Laval">
  Je suis fermé le week-end.
</College>
```

### Définition :

```jsx
const College = ({ categorie, endroit, children }) => (
  <div>{children}</div>
);
```

# <h1 id="etape12-state-intro">Étape 12 – Introduction à l’état (`state`) dans les classes</h1>

```jsx
class App extends Component {
  state = { cat: "public", lieu: "Gaspésie" };

  render() {
    return <College categorie={this.state.cat} endroit={this.state.lieu} />;
  }
}
```

---

# <h1 id="etape13-setstate">Étape 13 – Modifier le `state` avec un bouton (`setState`)</h1>

```jsx
changerCollege = () => {
  this.setState({ cat: "privé" });
};

render() {
  return (
    <>
      <College categorie={this.state.cat} endroit={this.state.lieu} />
      <button onClick={this.changerCollege}>Changer le collège</button>
    </>
  );
}
```
